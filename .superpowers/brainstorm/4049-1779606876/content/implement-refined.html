/implement 细化 — 加入任务拆解

编码前必须先拆解任务，编号追踪，注明影响范围

/implement — Implementation 编码实现
输入: 蓝图（Spec + 流程图 + 测试清单）

① 任务拆解 落盘
基于蓝图将需求拆解为编号开发任务
每个任务标注：大致影响的模块/文件、依赖关系、预估复杂度
② 技术架构文档
选择合适的 UML/泳道图/状态机图描述技术方案
数据流设计、接口定义、关键决策说明
③ 编码执行
无依赖任务 → 并行 Sub-agent + TDD
有依赖任务 → 顺序 Sub-agent + TDD
每任务：代码审查 + 提交
输出: 任务拆解清单 + 代码提交记录 → 触发 /verify

落盘文件总览

产出物 来源命令 编号追踪 跨会话恢复
需求清单 /breakdown ✅ R-001, R-002... ✅ 落盘
设计规格 + 业务流程图 /blueprint — ✅ 落盘
任务拆解清单 /implement ① ✅ T-001, T-002... ✅ 落盘
测试用例清单 /blueprint ③ ✅ TC-001, TC-002... ✅ 落盘
