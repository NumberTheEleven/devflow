Superpowers vs OpenSPEC — 工作流对比

两种主流 AI 开发规范工作流 + 你的方案框架

Superpowers 工作流
Phase 1: Brainstorming
探索 → 澄清 → 方案对比 → 设计确认 → Spec文档
↓
Phase 2: Writing Plans
Spec→Plan 拆解 → Bite-sized Tasks → 精确路径+代码
↓
Phase 3: Subagent-Driven Dev
逐任务 → 新Subagent → TDD → Spec审查 → 代码审查
↓
Phase 4: Verification & Finish
验证证据 → 测试 → Merge/PR/Keep/Discard
OpenSPEC 工作流
/opsx:propose
一键创建 proposal + design + tasks + specs
↓
/opsx:apply
AI 按 tasks.md 顺序实施，标记完成
↓
/opsx:archive
归档到 archive/ → 合并增量规范到 specs/
扩展命令: explore, new, continue, ff, verify, sync, onboard
