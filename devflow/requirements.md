## R-001: 在 verify Skill 中引入 AI 视觉审查机制

**优先级:** P0
**状态:** 待开始
**描述:** 当前 DevFlow verify 流程依赖 DOM 结构、控制台错误和网络健康来判定页面是否正常，无法识别 UI 重叠、覆盖、遮挡、错位、溢出等明显视觉缺陷。需要在 verify Skill 中增加基于 Playwright 截图 + AI 视觉分析的自动检测机制，把用户目前“手动截图发给 Claude 看”的 workflow 规范化、自动化。
**依赖:** 无
**验收标准:**
- [ ] `skills/verify/_SKILL.md` 中明确定义“视觉审查”的目标、触发条件和执行步骤
- [ ] 视觉审查仅在 TC 明确涉及前端/视觉/布局/样式时启用，无前端改动的功能不强制触发
- [ ] AI 视觉分析能识别并报告：元素重叠、内容被遮挡、布局错位、文字截断、异常空白、弹窗/抽屉未正确弹出等常见问题

## R-002: 扩展 L1 烟雾扫描支持视觉烟雾扫描

**优先级:** P0
**状态:** 待开始
**描述:** 在现有 L1（控制台错误、网络健康、运行时健康、DOM 快照）基础上，增加“视觉烟雾扫描”子步骤。对路由到 L1 且含视觉/布局关键词的 TC，在页面加载后调用 `browser_take_screenshot` 捕获截图，由 AI 进行视觉异常扫描。
**依赖:** R-001
**验收标准:**
- [ ] L1 流程中新增“视觉烟雾扫描”小节，说明截图时机、截图范围（全页或视口）、AI prompt 要求
- [ ] 视觉烟雾扫描的结果写入 verification-log.md 的“视觉问题清单”
- [ ] 视觉问题按严重程度分级（例如：阻塞 / 明显 / 轻微），并给出建议修复阶段

---

## R-003: 扩展 L2 交互验证支持交互前后截图分析

**优先级:** P0
**状态:** 待开始
**描述:** 当前 L2 仅对比操作前后的 DOM 结构变化，无法发现交互引发的视觉问题（例如点击按钮后弹窗被 mask 遮挡、下拉菜单被截断、toast 被导航栏盖住）。需要对路由到 L2 且含交互 + 视觉相关 TC，在每个关键动作前后截图，由 AI 分析视觉变化是否符合预期。
**依赖:** R-001
**验收标准:**
- [ ] L2 流程中新增“交互截图分析”步骤，明确何时截图（动作前、动作后、异步结果稳定后）
- [ ] 截图分析关注：新出现的 UI 是否完整可见、焦点元素是否被遮挡、动画/过渡是否导致布局跳动
- [ ] 若 AI 发现视觉异常，将该 TC 标记为 fail 或降级到 L3 人工复核，并记录证据

## R-004: 增加程序化布局断言作为 AI 视觉分析的补充

**优先级:** P1
**状态:** 待开始
**描述:** 为降低 AI 调用成本并提供确定性兜底，增加基于 Playwright 元素几何信息的程序化检查，用于捕获规则明确的布局问题。
**依赖:** R-001
**验收标准:**
- [ ] 定义至少 3 种通用布局断言：可见元素互相重叠、元素超出可视区域、文字/内容被容器截断
- [ ] 程序化断言在 L1/L2 中作为 AI 视觉分析的前置或并行步骤执行
- [ ] 断言失败的问题同样汇入“视觉问题清单”

---

## R-005: 调整 TC 路由规则，视觉/前端相关 TC 优先走 L1/L2

**优先级:** P1
**状态:** 待开始
**描述:** 当前路由规则把“样式、布局、视觉、美观”等关键词直接路由到 L3 手工验证，导致大量本可自动化的视觉问题依赖人工发现。需要调整路由规则，让明确涉及前端展示的问题优先由 L1/L2 的截图/断言覆盖，只有主观审美、动画体验、多角色权限等才保留在 L3。
**依赖:** R-002, R-003
**验收标准:**
- [ ] 更新 `skills/verify/_SKILL.md` 中的 TC 路由关键词表
- [ ] “重叠、遮挡、覆盖、溢出、截断、布局、错位、白屏、闪烁”等词优先映射到 L1/L2
- [ ] “美观、流畅、体验、动画、权限、角色”等仍需主观判断的词保留在 L3
