## TC-001: L1 烟雾扫描 4 类检查逐页执行

**状态:** 待开始
**覆盖:** R-001（L1 烟雾扫描层）
**类型:** 端到端
**步骤:**
1. 准备一个包含多个页面的项目（至少 3 个路由），其中一个页面故意有 console error
2. 指向项目的 baseURL
3. 启动 verify 流程，观察 L1 执行

**预期结果:** 对每个路由依次执行 console error / network health / runtime health / DOM snapshot 四类检查，结果逐页记录到 verification-log.md

## TC-023: 报告结论明确区分三种结果

**状态:** 待开始
**覆盖:** R-007（验证报告格式升级）
**类型:** 手工
**步骤:**
1. 分别运行三个场景：全部通过、部分通过（有 unverified）、不通过（有 FAIL）

**预期结果:**

- 不通过 → "❌ 验证不通过：X 项失败，详见失败清单"
